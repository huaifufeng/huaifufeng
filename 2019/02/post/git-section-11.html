别名

简单别名

有两种方法来创建Git别名：

  • 编辑~/.gitconfig文件：

    [alias]
ci = commit 
st = status 
co = checkout

  • 使用命令行：

    git config --global alias.ci "commit" 
git config --global alias.st "status" 
git config --global alias.co "checkout"

在别名创建之后：

  • git ci就是git commit
  • git st就是git status
  • git co就是git checkout

像正常的命令一样，别名也可以使用参数，例如：

 git ci -m "Commit message..." 
 git co -b feature-42

显示搜索别名

你可以使用--get-regexp选项来列出所有的别名：

$ git config --get-regexp '^alias\.'

搜索别名

想要搜索别名的话，在.gitconfig中添加下面的配置：

aliases = !git config --list | grep ^alias\\. | cut -c 7- | grep -Ei --color \"$1\" "#"

然后你就可以使用过下面的命令：

  • git aliases显示所有的别名
  • git aliases commit只显示包含commit的别名

高级别名

Git允许你在别名中使用非Git命令和完全的shell语法，如果你在命令的签名使用！。在你的~/.gitconfig文件中添加下面的内容：

[alias]
	temp = !git add -A && git commit -m "Temp"

这在签名的别名中提供了完善的shell语法，意味着可以使用完整的shell函数来构建更加复杂的别名，例如使用过命令行参数的别名：

[alias]
    ignore = "!f() { echo $1 >> .gitignore; }; f"

上面的别名定义了一个f函数，可以运行这个命令带上不同的参数。git ignore .tmp/将会添加.tmp/文件夹到.gitignore文件中。

实际上，这个模式非常有用，Git为您定义了$1、$2等变量，因此您甚至不必为它定义特殊的函数。（但请记住，Git无论如何也会附加参数，即使您通过这些变量访问它，所以您可能希望在末尾添加一个虚拟命令。）

注意带有！前缀的别名命令是在你迁出的项目的根目录执行的，即使你再代码树更深处的地方。这样在项目的根目录执行命令就不用显示的使用cd命令切换到具体的目录了。

 [alias]
 	ignore = "! echo $1 >> .gitignore"

临时忽略追踪文件

临时标记文件为忽略的（在别名后面使用文件名）：

unwatch = update-index --assume-unchanged

重新追踪文件使用：

watch = update-index --no-assume-unchanged

列出所有临时忽略的文件：

unwatched = "!git ls-files -v | grep '^[[:lower:]]'"

清除临时忽略的列表：

watchall = "!git unwatched | xargs -L 1 -I % sh -c 'git watch `echo % | cut -c 2-`'"

下面是这些别名的使用例子：

git unwatch my_file.txt 
git watch my_file.txt 
git unwatched
git watchall

分支图显示好看的日志

[alias]
  logp=log --pretty=format:'%h %ad | %s%d [%an]' --graph --date=short
  lg =  log --graph --date-order --first-parent \
     --pretty=format:'%C(auto)%h%Creset %C(auto)%d%Creset %s %C(green)(%ad) %C(bold cyan)<%an>%Creset'
  lgb = log --graph --date-order --branches --first-parent \
     --pretty=format:'%C(auto)%h%Creset %C(auto)%d%Creset %s %C(green)(%ad) %C(bold
cyan)<%an>%Creset'
 lga = log --graph --date-order --all \
   --pretty=format:'%C(auto)%h%Creset %C(auto)%d%Creset %s %C(green)(%ad) %C(bold
cyan)<%an>%Creset'

下面是--pretty选项使用的选项和占位符的含义介绍（git help log这个命令可以查看详细信息）：

--graph显示提交树。

--date-order可能的话按照提交的时间戳排序。

--first-parent在merge的提交上只显示第一级父

--branches显示所有的本地分支（默认只显示当前分支）

--all显示所有的本地和远端分支

%h显示hash值

%ad作者的日期戳

%an作者名称

%C(auto)使用[color]模块定义的颜色

%Creset重置这个颜色

%d装饰（分支和标记名称）

%s提交的信息

%ad作者日期（和--date选项效果一样，不是提交者日期）

查看.gitignore忽略的文件

[ alias ]
	ignored = ! git ls-files --others --ignored --exclude-standard --directory \ && git 		ls-files --others -i --exclude-standard

想要一行显示一个的话，可以使用grep命令（只是文件夹）：

$ git ignored | grep '/$' 
.yardoc/
doc/

或者统计文件数量：

$ git ignored | wc -l
199811   # oops, my home directory is getting crowded

在保持线性历史的同时更新代码

有时候你想要为你代码的提交保持一个线性的历史。如果你在一个分支上工作了一段时间，现在需要git pull一个分支到本地，这回创建一个merge提交。

 [alias]
  	up = pull --rebase

这将会更新上游的源，把所有本地未提交的工作放到你pull下了的工作的最开头。

可以使用：

git up

取消暂存文件

一般来说，想要删除等待提交的暂存文件使用git reset命令，reset根据传给他参数的不同，有很多的方法执行。为了完全取消对所有文件的暂存，我们可以创建一个不需要记忆传递什么参数的新的reset别名。

$ git config --global alias.unstage "reset --"

现在当你想要取消文件的暂存是，可以使用git unstag这个命令，这将会很好的执行。

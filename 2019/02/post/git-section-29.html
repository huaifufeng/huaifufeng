在Git修改，暂存和提交代码之后，接下来就需要push操作来把你本地的修改分享给其他人，把你本地的修改推送到仓库服务器。这个主题主要介绍如果使用Git来推送代码。

推送指定对象到远端分支

一般语法

git push <remottename> <object>:<remotebranchname>

例子

git push origin master:wip-yourname

这个例子将会推送你本地的master分支到origin的wip-yourname分支上面（一般是你克隆的仓库）。

删除远端分支

删除远端分支就等价于推送一个空对象到远端。

git push <remotename> :<remotebranchname>

例子

git push origin :wip-yourname

将会删除你的远端分支wip-yourname。

除了使用冒号，你可以使用—delete选项来代替，在一些情况下是更加可读的。

例子

git push origin --delete wip-yourname

推送单独提交

如果你本地有一个单独的提交，你想要推送到远端不推送其他的内容，你可以使用下面的命令：

git push <remotename> <commit SHA>:<remotebrranchname>

例子

假设存在一个像下面一样的提交历史：

eeb32bc Commit 1 - already pushed
347d700 Commit 2 - want to push
e539af8 Commit 3 - only local
5d339db Commit 4 - only local

只想要吧提交347d700推送到远端的master分支，使用下面的命令：

git push origin 347d700:master

push

git push

将会推送你的代码到存在的上游。根据git配置文件的不同，将会推送你当前分支的代码（默认在版本2.x），或者推送所有的分支（默认版本1.x）。

指定远端仓库

使用Git的时候，拥有多个远端仓库时非常方便的。想要指定一个要推送的远端仓库，只需要在命令后面添加仓库名称：

git push origin

指定分支

想要推送指定的分支feature_x：

git push origin feature_x

设置远端追踪分支

除非你工作的分支最初来源于远端仓库，否则你第一次使用git push推送代码不会有作用。你必须执行下面的命令来告诉Git将当前分支推送到特定的远端/分支组合。

git push --set-upstream origin master

在这个例子里，master分支是远端origin的分支名。你可以使用—set-upstream的简写-u。

推送到一个新仓库

想要推送到一个你还没有创建或者现在空的仓库内：

  1. 在GitHub上面创建一个仓库（如果可以的话）
  2. 复制你获得的url，像https://github.com/USERNAME/REPO_NAME.git这种格式的
  3. 在你的本地仓库执行git remote add origin url命令，添加远端仓库的地址
    • 验证添加的结果，使用git remote -v
  4. 使用git push origin master推送代码

现在你的代码被添加到GitHub上面了。

解释

推送代码意味着：分析本地提交和远端仓库代码的不同，然后把这些修改写入上游。当推送成功之后，你本地代码库和远端仓库时同步的，其他用户可以看到你的提交。

强制推送

有时候，当你本地的修改和远端修改不兼容的时候（当你没有跟进远端分支，或者远端分支不是你本地分支的祖先分支），唯一的办法就是你强制推送你的修改：

git push -f

或者：

git push --force

重要注意：

这将会覆盖远端的修改，然后你的远端将会和你的本地保持一致。

注意：使用这个命令将会造成远端仓库丢失提交。此外，强烈建议不要在与其他人共享远端仓库的情况下进行强制推送操作。因为其他人的历史记录中会保存每个被覆盖的提交，从而让他们的工作与远端仓库不同步。

根据经验，只在下面的情况下进行强制推送：

  • 除了你拉取了你要覆盖的提交，其他人都没有拉取
  • 您可以强制每个人在强制推送之后克隆一个新副本，并使每个人都将其更改应用于该副本（人们可能会因此而恨您）。

推送tag

git push --tags

推送所有在本地但不在远端仓库的git tags。

修改默认推送行为

Current 修改远端仓库中与当前分支共享名称的分支：

git config push.default current

Simple 推送到上游分支，如果上游分支被叫做其他名称的话就没有作用

git config push.default simple

Upstream 推送到上游分支，不管它叫什么名称

git config push.default upstream

Matching 将与本地和远程git config push.default匹配的所有分支向上推送。

在你设置首选样式之后，使用:

git push

来更新远端仓库。

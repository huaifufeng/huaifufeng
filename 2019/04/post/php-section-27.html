命名空间

声明命名空间

一个命名空间可以像下面这样：

  • namespace MyProject 声明一个命名空间MyProject
  • namespace MyProject\Security\Cryptography 声明一个嵌套的命名空间
  • namespace MyProject {…} 声明一个花括号包围的命名空间

虽然你可以在一个文件中声明多个命名空间，但是强烈建议一个文件只声明一个命名空间：

namespace First {
    class A { ... }; // Define class A in the namespace First.
}
namespace Second {
    class B { ... }; // Define class B in the namespace Second.
}
namespace {
    class C { ... }; // Define class C in the root namespace.
}

每当你声明了一个命名空间，你在之后定义的类都在这个命名空间里面：

namespace MyProject\Shapes;
class Rectangle { ... } 
class Square { ... } 
class Circle { ... }

一个命名空间可以在不同的文件中多次声明。上面的例子在一个单独的文件中定义了一个命名空间MyProject\Shapes，并且定义了3个类。更加可取的方式是将这个文件拆入3个单独的文件，每个文件都以namespace MyProject\Shapes;开头，并且只包含一个类定义。这在PSR-4中有详细的介绍。

引用命名空间类或方法

像上面介绍的，我们可以像下面这样声明一个类：

namespace MyProject\Shapes; 
class Rectangle { ... }

想要引用这个类，就需要使用完整的路径：

$rectangle = new MyProject\Shapes\Rectangle();

可以在使用的文件中使用use语句导入这个类，之后就可以使用短名称：

// Rectangle becomes an alias to MyProject\Shapes\Rectangle
use MyProject\Shapes\Rectangle; 
$rectangle = new Rectangle();

在PHP7.0之后，你可以在一个use语句中使用花括号组合多个选项：

use MyProject\Shapes\{
    Rectangle,  //Same as `use MyProject\Shapes\Rectangle`
    Circle,     //Same as `use MyProject\Shapes\Circle` 
    Triangle,   //Same as `use MyProject\Shapes\Triangle`
    
    Polygon\FiveSides,  //You can also import sub-namespaces
    Polygon\SixSides   //In a grouped `use`-statement
};

$rectangle = new Rectangle();

有时候两个类有相同的名称。如果这两个类在不用的命名空间，这是没有问题的。但是要导入这两个类的话就会出现问题了：

use MyProject\Shapes\Oval;
use MyProject\Languages\Oval; // Apparantly Oval is also a language! 
// Error!

这可以通过使用as关键字来定义类的别名：

use MyProject\Shapes\Oval as OvalShape;
use MyProject\Languages\Oval as OvalLanguage

想要引用当前命名空间之外的类，需要在命名空间前面加上\，否则将会在当前命名空间中进行相对命名空间的查找：

namespace MyProject\Shapes;

// References MyProject\Shapes\Rectangle. Correct!
$a = new Rectangle();

// References MyProject\Shapes\Rectangle. Correct, but unneeded!
$a = new \MyProject\Shapes\Rectangle();

// References MyProject\Shapes\MyProject\Shapes\Rectangle. Incorrect!
$a = new MyProject\Shapes\Rectangle();

// Referencing StdClass from within a namespace requires a \ prefix
// since it is not defined in a namespace, meaning it is global.

// References StdClass. Correct!
$a = new \StdClass();

// References MyProject\Shapes\StdClass. Incorrect!
$a = new StdClass();

子命名空间

要使用层次结构声明单个命名空间，请使用以下示例：

namespace MyProject\Sub\Level;

const CONNECT_OK = 1;
class Connection { /* ... */ }

function connect() { /* ... */ }

上面的例子创建了：

常量MyProject\Sub\Level\CONNECT_OK

类MyProject\Sub\Level\Connection

函数MyProject\Sub\Level\connect

什么是命名空间

PHP社区有很多开发者创建了很多的代码。这意味着PHP中一个仓库中拥有和另外仓库中相同的类名。当在一个命名空间中使用这两个仓库的时候，会发送冲突并导致问题。

命名空间就是用来解决这个问题的。就像PHP手册说的，命名空间可以和命名空间中文件在操作系统的目录进行比较，两个同名的文件可以在不同的目录中存在。同样，两个同名的PHP类可以在不同的命名空间中。

为代码增加命名空间是很重要的，这样其他开发人员就可以使用它，而不必担心与其他库发生冲突。

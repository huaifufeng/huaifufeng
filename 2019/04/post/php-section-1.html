web服务器输出HTML

PHP可以用来在HTML文件里面添加内容。当HTML被web服务器处理的时候，PHP的脚本将会被web服务器执行，然后结果的HTML内容将会被输出到浏览器。

下面的HTML代码包含了一个输出Hello World!的内容到HTML中的PHP代码：

<!DOCTYPE html>
<html>
    <head>
        <title>PHP!</title>
    </head>
    <body>
        <p><?php echo "Hello World!"; ?></p>
    </body>
</html>

当这些内容被保存到一个PHP的脚本，并且被一个web服务器执行之后。下面的HTML代码将会被发送给用户的浏览器：

<!DOCTYPE html>
<html>
    <head>
				<title>PHP!</title> 
		</head>
		<body>
				<p>Hello world!</p>
    </body>
</html>

PHP 5.x Version>=5.4

echo有一种短语法，可以让你打印相同的值。在PHP 5.4.0之前，这种短语法只有在short_open_tag开启的情况下才会生效。

例如，下面的代码：

<p><?= "Hello World!" ?></p>

它的输出和下面代码的输出是完全等级的：

<p><?php echo "Hello world!"; ?></p>

实际的应用中，所有由PHP输出到HTML页面的代码都有进行转义，用来防范XSS(跨站脚本工具)攻击或文本损坏。

也可以查看PSR-1：描述了正确的开发方式，包括使用短标签(<?= ?>)

Hello, World!

在PHP里面最常用的打印输出的方法是echo:

echo "Hello, World!\n";

另外，你可以使用print:

print "Hello, World!\n";

这两个语句实现了相同的功能，但有一些细微的区别：

  • echo没有返回值，但是print有一个int型的1返回值
  • echo不加括号时可以有多个参数，但是print只能有一个参数
  • echo比print稍微快一些

echo和print都是语言结构，而不是一个函数。这意味着它们不需要为参数加上括号。当然也可以和函数一样，使用括号。很多地方都有使用echo和print的地方。

和C类似的printf方法也是支持的，像下面这个例子：

printf("%s\n", "Hello, World!");

有关在PHP中输出变量的全面介绍，可以在后面的”输出变量的值”中详细查看。

web服务器输出非HTMl内容

有时候，需要修改web服务器的默认内容格式。例如，在某些情况下，你可能需要输出纯文本，JSON，或者XML等等。

header()函数可以发送一个原生的HTTP头。你可以添加Content-Type头来通知浏览器我们发送内容的格式。

看下面的代码，我们发送的头Content-Type是text/plain格式的内容：

header("Content-Type: text/plain");
echo "Hello World!";

这将会产生一个包含下面内容的纯文本文件：

Hello World!

要产生一个JSON文件，可以使用application/json格式的文件内容头：

header("Content-Type: application/json");

//创建一个PHP数组
$data= ["response" => "Hello World"];

//转换生成一个有效的JSON串
echo json_encode($data);

这将会产生一个包含下面内容的application/json格式的文件：

{“response”:“Hello World”}

注意：header()方法必须在PHP输出任何内容之前，或者已经发生响应头之前执行。下面的代码：

//错误：我们不能在header方法之前发送内容
echo "Hello";

//header必须在其他输出之前执行
header("Content-Type: text/plain");
echo "World";

这将会产生一个报警：

Warning：不能修改头信息-头信息已经发送了(输出产生在/dir/example.php:2) /dir/example.php 第3行

当使用header()函数的时候，它必须是第一个从服务器发送的字节。因为这个原因，在PHP开始符<?php之前不能有空行或者空格。同样的原因，在PSR-2中，省略PHP的结束符?>，使用PHP的代码作为文件的结尾是一种非常号的编程方式。

可以查看”输出缓存”章节，来学习怎么把你的内容放入一个变量，之后在输出这些内容，例如在header方法之后。

PHP内建服务器

PHP 5.4+开始提供一个开发用的服务器。这可以被用来运行代码而不必按照生存使用的服务器，如nginx和Apache。这个内建服务器是为了开发和测试建立的。

可以通过-S选项来开启这个服务器：

php -S host/ip:port

示例：

  1. 创建一个包含下面内容的index.php文件：

    <?php
echo "Hello World from built-in PHP server";

  2. 在命令行中运行php -S localhost:8080这个命令。记住不要包含http://。这将开始一个监听8080端口，以当前目录为根目录的服务器。

  3. 打开浏览器，输出localhost:8080。你就可以在浏览器中开的Hello World。

配置

要替换默认的文件根目录(例如：当前的目录)。可以使用-t选项：

php -S <host/ip>:<port> -t <directory>

例如：你有一个public/目录，你可以使用下面的命令把public目录作为服务器的文件根目录：

php -S localhost:8080 -t public/

Logs

开发服务器的每次请求，都会在命令行下面输出下面的内容：

[Mon Aug 15 18:20:19 2016] ::1:52455 [200]: /

PHP命令行

PHP也可以直接使用CLI(命令行接口)来运行命令。

CLI基本上和PHPweb服务器执行过程，除了输入和输出的不同。

触发

PHP的CLI允许你用四种方式执行PHP：

  1. 标准输入。运行没有参数的PHP命令，但是使用管道执行：echo '<?php echo "Hello World!";' | php
  2. 把文件名作为参数。执行php命令，其中php源码文件是第一个参数：php hello_world.php
  3. 把代码作为参数。使用-r选项执行php命令，后面是要执行的代码。其中<?php不是需要的，任何参数都被认为是PHP代码。php -r 'echo "Hello World!"';
  4. 交互模式。使用-a选项来开启一个交互的shell。然后输入php代码，按下return执行。$ php -a开启交互。php > echo "Hello World!"

输出

所有那些用来在web服务器产生输出的函数和控制都可以用来进行标准输出（文件标识符1），所有在web服务器产生错误的操作都会产生错误输出(文件标识符2)。

Example.php
<?php
echo "Stdout 1\n";
trigger_error("Stderr 2\n"); print_r("Stdout 3\n");
fwrite(STDERR, "Stderr 4\n");
throw new RuntimeException("Stderr 5\n"); 
?>
Stdout 6
Shell命令行
$ php Example.php 2>stderr.log >stdout.log;\
> echo STDOUT; cat stdout.log; echo;\
> echo STDERR; cat stderr.log\

指令分离

像大多数类C的语言一样，没一个语句都是用一个分号结尾。当然，一个结束符也可以作为PHP代码的结束标志。

如果PHP代码最后使用分号结尾，代码结束符是可选的，如果之后已经没有代码了。例如，在下面的代码中，我们可以省略echo "No error";后面的结尾符。

<?php echo "No error"; //没有结束符

然而，如果PHP代码块之后还有其他的代码，这里的结束符是不能省略的：

<?php echo "This will cause an error if you leave out the closing tag"; ?> 
<html>
		<body>
    </body>
</html>

如果PHP代码块的最后有一个结束符，我们可以省略末尾的分号：

 <?php echo "I hope this helps! :D"; 
 echo "No error" ?>

通常推荐在每一个语句后面使用分号，处理后面没有代码之外，每一个PHP代码块都使用结束符。

所以你的代码基本是下面这样的：

<?php
    echo "Here we use a semicolon!";
    echo "Here as well!";
    echo "Here as well!";
    echo "Here we use a semicolon and a closing tag because more code follows";
?>
<p>Some HTML code goes here</p> <?php
    echo "Here we use a semicolon!";
    echo "Here as well!";
    echo "Here as well!";
    echo "Here we use a semicolon and a closing tag because more code follows";
?>
<p>Some HTML code goes here</p> <?php
    echo "Here we use a semicolon!";
    echo "Here as well!";
    echo "Here as well!";
    echo "Here we use a semicolon but leave out the closing tag";

PHP标记

有3中标记来在一个文件中声明PHP代码块。PHP语法分析器寻找开始和结束符(如果存在)，来同时解释器什么代码是需要解析的。

标准标记

这些标签是文件中嵌套PHP代码的标准方法：

<?php
    echo "Hello World!";
?>

PHP 5.x Version>=5.4

echo标记

这些标签是所有PHP版本都有效的，但是从5.4开始是默认开启的。在以前的版本中，echo标签总是和短标签一起使用：

<?= "Hello World" ?>

短标记

你可以使用short_open_tag来开启和关闭短标签：

<?
		echo "Hello World!";
?>

短标签：

  • 在所有主要的PHP代码规范中是不被允许的
  • 在官方文档中是阻止使用的
  • 在大多版本中是默认禁止的
  • 干扰内部的XML处理指令
  • 在大多数开源项目中不被接受

PHP 5.x Version <= 5.6

ASP标记

通过开始asp_tags选项，可以使用ASP格式的标签：

<%
    echo "Hello World!";
%>

这是一个历史性的问题，不应该使用。并且它们在PHP7.0中被移除了。

创建你第一个Java程序

使用你的文本编辑器或IDE创建一个名称为HelloWorld.java的文件。然后把下面的代码贴入那个文件中，并且保存文件：

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, Wolrd!");
    }
}

注意：对应Java，将会识别这个文件public class(不会抛出编译时错误)，这个文件名必须和类名是相同的(这里就是HelloWorld)，扩展名为.java。这个类的前面需要public标识符。

命名约定推荐Java的类以大写字母开始，然后使用驼峰命名格式来命名(就是除了第一个字符之外，单词的首字母大写，其他小写)。这个约定是和下划线和$符相对的。

接下来是编译代码，打开一个终端窗口，切换到HelloWorld.java文件所在的目录：

cd /path/to/containing/folder/

注意：cd是用来切换目录的终端命令

$ javac HelloWorld.java

通常会遇到一个错误'javac' is not recognized as an internal or external command, operable program or batch file，即使你已经安装了JDK并且可以在IDE(eclipse等)中执行。因为命令的路径默认没有添加到环境变量内。

因此当你在windows环境遇到这个问题，首先就是查找你的javac.exe的路径。很可能是在C:\Program Files\Java\jdk(version number)\bin，然后在下面这样运行：

 $ C:\Program Files\Java\jdk(version number)\bin\javac HelloWorld.java

我们想上面代码一样调用javac程序。因为这样你的操作系统才会知道javac所在的位置。现在我们介绍另外的方式，这样就不行填写整个程序路径了，那就是环境变量PATH。

在Windows XP/Vista/7/8/10上面修改PATH环境变量：

  • 控制面板=>系统=>高级系统配置
  • 切换：”高级”tab=>环境变量
  • 在”系统环境变量”中，选择”PATH”=>修改

这修改了就没法测回，你要小心一些。首先把路径复制到编辑器上。然后获得javac所在目录的目录地址，并复制这个地址。这将会是c:\Program Files\Java\jdk1.8.0_xx\bin这样的。

在”变量值”字段中，添加这个路径在之前存在路径的前面，然后添加分号。不要删除原有的值。

Variable name  : PATH
Variable value : c:\Program Files\Java\jdk1.8.0_xx\bin;[Existing Entries...]

现在就可以了。

对于Linux系统，查看这里。

注意：javac调用java的编译器。

这个编译器将会产生一个叫做HelloWorld.class的字节码文件，这个文件可以被JVM执行。Java的编译器javac解析有Java代码写的文件，然后生成一个字节码的类文件。另外，编译器还可以使用注释处理API处理源码文件和类文件中的注释。编译器是命令行工具，也可以使用Java编译器API调用。

然后使用java运行包含main方法的类文件(这里是HelloWorld)，这里忽略.class后缀：

$ java HelloWorld

注意：这里java命令运行了一个Java应用。

这将会在你的终端输出：

Hello, World!

你成功的进行了代码编写，并且实现了你第一个Java程序。

注意：为了保证上面的命令(java，javac等)正确执行，你必须确保：

  • JDK已经安装了(Orcale，OpenJDK，或者其他的)
  • 环境变量都正确设置了

你将需要JVM提供的编译器javac和执行器java。想要知道自己安装的是什么版本的，可以使用命令java -version和javac -version来查看。这将会在终端中输出你使用的版本号(1.8.0_73)

详细解释Hello World

这个Hello World程序只包含一个文件，这个文件包含了一个HelloWorld类定义，一个main方法和main方法内的一条语句。

public class HelloWorld {

这里class关键字在类定义的开始，后面跟着类的名称HelloWorld。每一个Java程序至少包含一个类定义(详细信息查看类定义部分)。

public static void main(String[] args) {

这是一个入口方法(注意这个格式public static void main(String[]))，这个方法是给JVM运行这个程序用的。每一个Java程序都必须有一个。这个方法：

  • public : 意味着这个方法可以在程序外被调用。查看可见性部分。
  • static ：这个方法是类级别的，可以被类直接调用
  • void：意味着这个方法没有返回值。注意：不像C和C++返会一个整数码，Java这里是System.exit()

这个main方法接收：

  • 一个字符串数组(被叫做args)作为参数(也是命令行参数)

这机会是所有的Java入口方法必须的。

不要求的部分：

  • 这个名称args是一个可选的变量名称，所以你可以叫任何你希望的名字，尽管通常被称作args
  • 这个方法的参数是数组(String[] args)还是Varargs(String… args)都没有关系，因为数组可以被传递个Varargs

注意：一个应用可以有多个类定义了入口方法main。应用实际使用的如果方法根据java命令后面的类名决定。

在main方法里面，我们看到下面这条语句：

System.out.println("Hello, World!");

让我们把这个语句拆开来一个个分享：

元素 目的
System 这表明后面的表达式是在System上调用的，这个类在java.lang包里面
. 这是.操作符。这个操作符给你提供访问类元素(类字段和方法)的方法。因此，这个点操作符允许调用System类的out字段。
out 这是System类中包含的标准输出功能的静态字段PrintStream类型的名称
. 点运算符。这个点运算符提供访问out变量println方法
println 这是PrintStream类的一个方法名。这个方法输出参数内容到console中，并且打印一个换行符
( 这个括号表明是在访问一个方法(不是变量)，后面是传递个println方法的参数
“Hello World!” 这个是传递给方法println的字符串字面量参数。双引号表明这些字符是一个字符串
) 这个括号表示传递给println方法的参数已经结束了
; 分号表示语句的结尾。

注意：每一个Java的语句都要以分号结尾。

下面是方法体和类主体的结束部分：

 } // end of main function scope
} // end of class HelloWorld scope

下面是另一个介绍面向对象的例子。我们创建一个包含一个成员的球队。这里可以有很多球员，但是当我们讨论数组的时候在这样做。

第一步，我们定义一个Team类。

public class Team {
    Member member;
    public Team(Member member) {
        this.member = member;
    }
}

现在，我们定义Member类：

class Member {
    private String name;
    private String type;
    private int level; //注意这里的类型
    private int rank;
    
    public Member(String name, String type, int level, int rank) {
        this.name = name;
        this.type = type;
        this.level = level;
        this.rank = rank;
    }
}

为什么这里我们使用private呢？就如同，一个人想要知道你的名字，他应该直接问你，而不是那你的钱包看你的身份证。这里private就是做这种事情的，它避免从外部访问你的变量。你可以只通过getter这类方法来返回private变量。

现在我们把所有的代码放到一起，加上上面说的getter方法和main方法：

public class Team { 
    Member member;
    public Team(Member member) { 
        this.member = member;
    }
    // here's our main method
    public static void main(String[] args) {
        Member myMember = new Member("Aurieel", "light", 10, 1); 
        Team myTeam = new Team(myMember); 
        System.out.println(myTeam.member.getName());
        System.out.println(myTeam.member.getType()); 
        System.out.println(myTeam.member.getLevel()); 
        System.out.println(myTeam.member.getRank());
    } 
}

class Member {
    private String name; 
    private String type; 
    private int level; 
    private int rank;
    public Member(String name, String type, int level, int rank) { 
        this.name = name;
        this.type = type;
        this.level = level;
        this.rank = rank; 
    }
    /* let's define our getter functions here */ 
    public String getName() { 
        // what is your name?
        return this.name; // my name is ... 
    }
    public String getType() { // what is your type? 
        return this.type; // my type is ...
    }
    public int getLevel() { // what is your level? 
        return this.level; // my level is ...
    }
    public int getRank() { // what is your rank? 
        return this.rank; // my rank is
    }
}

输出：

Aurieel
light
10
1

在强调一下，Test类中的main方法是我们程序的入口。如果没有main方法，我们就没有办法通知JVM从哪里开始运行我们的程序。

1 - 因为HelloWorld和System类关系很小，所有只能访问public的数据。

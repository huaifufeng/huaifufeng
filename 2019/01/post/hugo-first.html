Hugo+GitHub搭建自己的blog

Posted by 怀府风 on Tuesday, January 1, 2019

TOC

    • 环境搭建
      • Hugo环境搭建
      • GitHub环境配置
    • 更进一步

2019年新的开始，新年第一天的首个工作就是把blog搭了起来，之后就要坚持做下去了，保证一周能有一篇有货的文章，这个也是给自己提的一个邀请，每周要学习至少一个知识点，可以分享的知识点。

这个blog是用hugo搭建的，因为最近在学习go语言，所以就挑选了这个以go语音实现的静态网站生成工具，在以后就要看下hugo的源码了，希望能够学习并升级这个工具。下面就介绍一下hugo+github搭建自己的blog。

环境搭建

这里涉及到两个环境的配置Hugo静态网站生成器和GitHub代码托管网站的配置，下面分别介绍两个环境的配置过程。

Hugo环境搭建

Hugo是一个用golang语言实现的一个静态网站生成器，不只是用来生成blog，还可以用来生成商城，信息网站等。它最主要的特性就是快，安全。这里就介绍下安装方法。

Hugo支持目前市面上的操作系统，包括：Mac，Windows，Linux，OpenBSD，FreeBSD。我这里用的Mac电脑，就介绍下Mac的安装，其他的环境可以到Hugo安装页面查看相应的安装步骤。

1、首先确定你的电脑是否安装有Homebrew，这是一个mac的安装包管理工具，可以用来安装软件。

brew --version

如果输出了版本号，说明你的电脑已经安装了这个软件，否则就需要执行下面的命令进行安装。

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

然后重新输入上面的命令进行安装。

2、使用Homebrew安装Hugo

brew install hugo

安装完毕之后使用下面的命令验证：

hugo version

获取到版本号说明安装成功。

3、然后就可以使用过hugo的命令来生成网站，生成页面了。

首先生成网址，先到达你存放网站信息的地方，使用下面的命令生成网站的根目录文件夹example(替换为你自己的项目目录)：

hugo new site example

生成网站的目录结构如下：

.
├── archetypes
├── assets
├── config.toml   //网站的配置文件
├── content       //网站内容的原始文件
├── data
├── layouts
├── static
└── themes    //网站的模板文件

其他的我们暂时不关心，目前用到的是content，themes，config.toml

首先需要把我们的项目变成一个git项目，才可以进行下面的操作：

git init
git add .
git commit -m 'init'

Hugo目前已经有了一批的模板，只需要到这个地址，就可以找到需要的模板了，我这个blog就是从中使用的模板。使用下面的命令就可以把一个模板下载到本地：

git submodule add https://github.com/panr/hugo-theme-hello-friend.git themes/hello-friend

修改config.toml文件，配置里面使用的模板：

baseUrl = "https://example.com/"
languageCode = "en-us"
title = "Your site title" 
theme = "hello-friend"    #使用的模板的文件夹名

然后在项目的根目录中执行

hugo

然后就会增加public/目录，也就是我们之后要放到github的目录。执行下面的命令，就可以看到网站的情况了：

hugo server -D

在浏览器中打开localhost:1313，就可以看到网站的首页了。

因为我们还没有写任何东西，所以网站是空荡荡的。让我们增加第一个信息。

在content/目录下可以新建新的目录，也可以不新建目录，我们在content下新建posts目录，用存放文件。

cd content
mkdir post
vi hello.md     //这里的内容需要使用markdown格式

在hello.md中增加如下的内容：

+++
title = "Hello Friend"
date = "1986-09-17"
author = "Elliot"
cover = "hello.jpg"
description = "\"Hello, friend?\" That's lame. Maybe I should give you a name?"
+++

> "Hello, friend?" That's lame.
> Maybe I should give you a name...  
> But that's a slippery slope.  
> You're only in my head.  
> We have to remember that...  
> Shit.  
> It's actually happened.  
> I'm talking to an imaginary person.
>
> **— Mr. Robot S01E01**

然后保存这个文件，在项目的根目录执行上面的命令：

hugo server -D

然后就可以看到网站的首个页面了：

自此，我们的网站就搭建起来了。但是这个网站目前只能在本地访问，也就是通过hugo server -D，然后在浏览器中通过localhost:1313访问。在github和gitee上面，提供了免费的静态网页访问支持，我们可以把我们的网站放到上面，这样就可以让其他人访问了。

这里挑选的是github，接下来就看下如果配置github，让其他人可以访问我们的网站。

GitHub环境配置

github是一个代码托管网站，可以在上面放自己的代码，但是目前免费的只支持公共版本，不支持私有版本。首先就是在上面注册一个账号，这个就不说了。

为了便于使用git命令访问自己的项目，需要在个人配置中增加sshKey，这个这里不细说了，百度一下就可以。

然后就可以新建一个项目了，我们这里的项目命名为和自己的用户名一，或者命名为你喜欢的其他名称也可以。

现在需要把我们本地的git项目和github这个新项目关联起来，把我们的项目代码放置到github上面去，从项目中获取项目的地址 git@gitee.com:example/example.git，用下面的命令绑定项目：

# 因为在github上面想要访问的话，只需要public文件夹，所以我们在public文件夹当单独建一个项目
cd public
git init
git add .
git commit -m "init"
git remote set-url origin git@gitee.com:example/example.git
git push origin master

这样我们本地修改之后，只需要把代码发布到github上面就自动可以访问了。

然后在项目的settings中可以配置网站的各种信息，如下图：

这里我们直接选择master分支就可以，因为我们使用的是hugo，所以目标这里就不能选择了。在往下的custom domain就是可以配自己的域名，在这里填入你自己的域名，比如www.blog.com，然后在你自己的域名解析工具内，解析你的域名到example.github.io上面，就可以通过你的域名访问了。

更进一步

我们整体的环境已经搭建完成了，也绑定了自己的域名，接下来就可以在content中添加自己的内容，在发布到github上面就可以了。github上面的网站访问只使用了public文件夹，但是我们其他的文件夹就不要了吗？不，这里推荐还是在建一个项目，把这个根项目放到保存起来，避免我们的电脑如果处理文件，就找不回来原来的内容了。

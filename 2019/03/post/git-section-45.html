git远端

参数 详情
-v, —verbos 冗长模式运行
-m 设置远端分支的头
—mirror=fetch 引用将不会保存在efs/remotes，而是保存在本地仓库内
—mirror=push git push表现的像--mirror被设置了
—no-tags git fetch <name>不会从远端仓库导入tag
-t 指定远端只追踪
-f 远端设置之后立即执行git fetch <name>
–tags git fetch <name>从远端仓库导入每个tag
-a, —auto 符号引用的HEAD设置为远端HEAD的相同的分支。
-d, —delete 所有列出的引用都会从远端仓库删除
–add 将添加到当前跟踪的分支列表（设置分支）
–add 不是修改某些URL，而是添加一些URL(设置URL)
–all 推送所有的分支。
–delete 所有匹配的将会被删除。
—push 推送URL而不是拉取URL
-n 远端HEAD不是首先使用git ls-remote <name>，而是使用缓存信息
—dry-run 报告将修剪哪些分支，但不要实际修剪它们
—prune 删除没有本地对应的远端分支

显示远端仓库

使用git remote列出所有配置的远端仓库。

显示你配置的每一个远端仓库的简单别名。

$ git remote 
premium 
premiumPro 
origin

想要显示更加详细的信息的话，使用—verbose或-v分支中。这将会包括URL和远端的类型(push和pull)。

$ git remote -v
premiumPro    https://github.com/user/CatClickerPro.git (fetch)
premiumPro    https://github.com/user/CatClickerPro.git (push)
premium       https://github.com/user/CatClicker.git (fetch) 
premium       https://github.com/user/CatClicker.git (push)
origin        https://github.com/ud/starter.git (fetch) 
origin        https://github.com/ud/starter.git (push)

修改仓库的远端地址

如果迁移了远端的仓库，你可能就想要做这个了。修改远端仓库的命令是：

git remote set-url

这个命令需要两个参考：一个存在的远端名称(origin，upstream)和url。

查看本地当前的远端url：

git remote -v
origin https://bitbucket.com/develop/myrepo.git (fetch) 
origin https://bitbucket.com/develop/myrepo.git (push)

修改你的远端url：

git remote set-url origin https://localserver/develop/myrepo.git

然后再次检测你的本地远端url：

git remote -v
origin https://localserver/develop/myrepo.git (fetch) 
origin https://localserver/develop/myrepo.git (push)

移除远端仓库

删除名为<name>的远端仓库。所有远端仓库追踪的分支和配置选项都将被删除。

删除远端分支dev：

git remote rm dev

添加远端仓库

在你本地仓库的根目录，使用git remote add来添加一个远端仓库。

git remote add <name> <url>

添加远端仓库的话使用一个简短名字<name>和git仓库的url。

git fetch <name>这个命令可以用来创建和更新远端追踪的分支<name>/<branch>。

显示远端仓库的详细信息

你可以使用git remote show <remote repository alias>来显示远端仓库的详细信息。

git remote show origin

将会输出：

remote origin
Fetch URL: https://localserver/develop/myrepo.git 
Push  URL: https://localserver/develop/myrepo.git 
HEAD branch: master
Remote branches:
    master      tracked
Local branches configured for 'git pull':
    master      merges with remote master
Local refs configured for 'git push':
master pushes to master (up to date)

重命名远端仓库

可以重命名远端仓库，从<old>到<new>。所有远端仓库追踪的分支和配置项都会被更新。

远端仓库命名为dev修改为dev1。

git remote rename dev dev1

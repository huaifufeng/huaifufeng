使用

修改引用中的对象名称

语法

git update-ref [-m <reason>] (-d <ref> [<oldvalue>] | [--no-deref] [--create-reflog] <ref> <newvalue> [<oldvalue>] | --stdin [-z])

普通语法

  1. 取消符合引用，修改分支头到新的对象
   git update-ref HEAD <newvalue>
  1. 验证引用中当前只和oldvalue匹配后，修改newvalue到引用中
   git update-ref refs/head/master <newvalue> <oldvalue>

上面的语法只有咱当前的值是oldvalue的时候，修改master分支的头到newvalue。

在验证引用还包含<oldvalue>之后，使用-d标识删除到命名的引用<ref>。

使用—create-reflog之后，更新引用将会为每个引用创建一个reflog，即使通常不会创建reflog。

使用-z标志以nul终止格式指定，该格式具有更新、创建、删除、验证等值。

update

如果提供了，在验证<oldvalue>之后，修改引用<ref>的值为<newvalue>。指定零值<newvalue>确保引用更新之后不存在，或者一个零值<oldvalue>确保引用更新之前不存在。

create

验证不存在之后使用<newvalue>创建<ref>。这个<newvalue>不能为0

delete

如果提供，验证<oldvalue>存在之后，删除<ref>。这时，<oldvalue>不能是0

verify

使用<oldvalue>验证<ref>中存在，但是不做修改。如果<oldvalue>为0或缺失，引用必须不存在。
